# Source Documentation Policy

## Title Guidelines

- Court orders: "[Court] Verdict/Order in Case [Case Number] on [Date]"
- Government documents: "[Issuing Body] [Document Type] dated [Date]"
- Media: "[Publication Name] - [Headline]"

## Description Requirements

The description must describe what the underlying content **is** (the document itself), not what a caseworker did or thinks.

**Good descriptions:**
- "Special Court verdict convicting Ram Sharma under Prevention of Corruption Act for procurement fraud at the Ministry of Education"
- "CIAA investigation report into irregularities in the Kathmandu-Terai Fast Track contract"
- "Kantipur news article reporting on the arrest of a government official"

**Bad descriptions:**
- "We found this document and it shows corruption"
- "Important evidence for our case"
- "Read this to learn more"

## Publication Date

- Required for MEDIA_NEWS sources
- Use BS dates for Nepal government documents
- Use AD dates for international sources
- If exact date unknown, use approximate date with notation

## File Requirements

- PDFs should be OCR'd if scanned
- Images should have clear resolution
- Never upload password-protected files
- Maximum file size: 5 MB per attachment

## Legal Notes

- All accused persons are presumed innocent until proven guilty
- Do not describe an accused as "corrupt" or "guilty" in source descriptions
- Use neutral language: "alleged," "accused of," "charged with"
